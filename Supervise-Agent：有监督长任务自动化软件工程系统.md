# Supervise-Agent：有监督长任务自动化软件工程系统

## 一、方案简介（Overview）

Supervise-Agent 是一套面向 AI 编程场景的自动化软件工程流水线系统，通过引入“分层裁决 + 多Agent协同 + 工程约束”，解决当前大模型在长任务执行中不稳定、不可控、不可验证的问题。

本方案不依赖昂贵的闭源模型（如 GPT / Claude），而是：

利用低成本开源模型 + 工程化控制系统，实现接近高端模型的稳定性与质量

## 二、问题定义（Why）

当前 AI 编程工具（如 OpenClaw / Claude Code / OpenCode）存在三个核心瓶颈：

### 1. 长任务失控（Long-task Drift）

# 模型在多轮执行中逐渐偏离目标

# 上下文污染严重（memory drift）

### 2. 缺乏可验证性（Unverifiable）

# 生成结果无法自动判断“是否正确”

# 依赖人工 review

### 3. 成本与质量矛盾（Cost vs Quality）

# 高质量模型（Claude / GPT）成本高

# 低成本模型（GLM / Kimi）能力不足

### 核心问题总结

不是“模型不会做”，而是“缺乏约束与收敛机制”

## 三、哲学设计思想（Philosophy）

Supervise-Agent 基于以下核心理念：

### 1. 约束 > 能力

# 与其不断提升模型能力，不如构建**外部约束系统**

### 2. 生成 ≠ 正确

# 所有生成结果必须经过**验证（Test）+ 对齐（Spec）**

### 3. 去中心化认知（De-centralized Cognition）

# 不依赖单一“大脑”

# 多Agent独立判断，避免错误放大

### 4. 软件工程化 AI

将 AI 编程问题转化为：

# 测试驱动（Test-Driven）

# 持续集成（CI-like）

# 自动修复（Generate-Repair）

## 四、核心创新点（Innovation）

### 创新1：分层裁决机制（Layered Decision）

A（生成） → C（测试裁决） → B（需求裁决）

C（Tester）：硬约束（是否正确运行）

B（Analyzer）：语义约束（是否符合需求）

实现“行为正确 + 需求对齐”双重保证

### 创新2：无状态监督（Stateless Supervision）

B / C 每轮重新分析

不共享历史 memory

防止错误累积与认知污染

### 创新3：差异驱动收敛（Diff-Driven Convergence）

系统通过“差异（Diff）”驱动迭代：

发现问题 → 生成修复 → 再验证 → 收敛

### 创新4：低成本模型放大机制

用“多次低成本推理 + 工程约束”
替代“单次高成本推理”

## 五、系统架构（Architecture）

                 ↓

        [存在差异 → 回A]
                 ↓
              DONE

## 六、核心模块（ABC模型）

### A：Builder（构建Agent）

负责代码生成与修复

支持长任务执行（可带memory）

### C：Tester（测试Agent）

基于需求生成测试用例

执行验证（功能 / 边界 / 异常）

👉 第一层裁决（硬约束）

### B：Analyzer（分析Agent）

对比需求与实现差异

输出结构化 Diff

👉 第二层裁决（语义约束）

## 七、控制机制（Controller）

### 决策逻辑

if 测试失败:
    修复功能
elif 存在需求偏差:
    修复需求差异
else:
    完成

### 收敛机制

最大迭代次数限制

多轮失败触发重构（Replan）

## 八、系统优势（Value）

### 相比单Agent

更稳定（不漂移）

可收敛（有终止条件）

可验证（测试驱动）

### 相比 Claude Code

成本低（开源模型）

可控性强（工程规则）

可扩展（模块化）
